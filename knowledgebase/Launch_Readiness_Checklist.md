OquWay Launch Readiness Checklist

Use this checklist before a class or school-year pilot goes live.

1. Student access

- Every student has an active profile.
- Every student can log in through the intended student login flow.
- Students only see their own assigned courses and progress.
- At least one test student per class has completed a full course path.

2. Teacher access

- Every teacher has an authorized Teacher Dashboard login.
- Every teacher can see only assigned classes, students, and course assignments.
- Password reset flow has been tested for at least one teacher.

3. Course readiness

- Every launch course has a clear title, description, and audience.
- Every launch course has at least one usable module.
- Every launch module has playable practice steps.
- No module contains empty placeholder steps.
- Teacher-reviewed external tasks have clear instructions and expected proof.
- Student preview has been tested for every launch module.

4. Daily classroom workflow

- Teacher can take attendance for each assigned class.
- Teacher can save and reload attendance for the selected date.
- Needs Help list shows students with review, activity, or progress concerns.
- Student detail opens from student rows.
- Review Queue shows student, class, course, module, task, submission, file proof, status, and feedback.

5. Feedback loop

- Student can submit an external task.
- Teacher can mark work complete, needs work, or incomplete.
- Teacher can leave feedback.
- Student can see the teacher feedback.
- Student can resubmit when work needs revision.

6. Launch data

- Real classes are created.
- Real students are assigned to classes.
- Real teachers are assigned as primary or assistant teachers.
- Courses are assigned to classes or individual students.
- One staff member owns account fixes and roster updates during launch week.

7. Smoke testing

- Student login smoke passes.
- Student dashboard load smoke passes.
- Course open/progress smoke passes.
- External task submission smoke passes.
- Teacher review queue smoke passes.
- Attendance load/save smoke passes.

Launch rule: do not add new feature scope during the final week unless it fixes a launch-blocking issue.
